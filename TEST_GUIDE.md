# 更新功能测试指南

## 快速测试步骤

### 1. 启动程序
```bash
python algorithm_platform.py
```
或在Windows上双击 `start.bat`

### 2. 测试一键配置设备
1. 点击工具栏的 **"🚀 一键配置设备"** 按钮
2. 输入WiFi名称和密码
3. 点击 **"开始配置"**
4. 等待配置完成（约1-2分钟）
5. 验证：
   - ✅ 状态栏显示设备IP（绿色）
   - ✅ RTSP地址显示在工具栏
   - ✅ MQTT自动连接成功

### 3. 测试模型与配置页面（新合并页面）

#### 3.1 推送模型
1. 切换到 **"模型与配置"** 标签页
2. 在"模型文件管理"区域，点击 **"浏览..."** 选择RKNN模型文件
3. 点击 **"📤 推送模型到设备"**
4. 确认推送
5. 等待推送完成
6. 验证：
   - ✅ 弹出成功对话框，提示 multi_media 进程已重启
   - ✅ 提示用户进行参数配置
   - ✅ 模型列表自动刷新，显示新模型

#### 3.2 删除模型
1. 在"已部署模型"表格中，找到要删除的模型
2. 点击该行右侧的 **"🗑️ 删除"** 按钮
3. 确认删除操作
4. 验证：
   - ✅ 模型从列表中消失
   - ✅ 设备上对应文件已被删除

#### 3.3 配置参数
1. 在"配置文件管理"区域，点击 **"浏览..."** 选择配置文件（JSON或INI）
2. 点击 **"📂 加载配置"**
3. 在配置编辑器中修改参数
4. 点击 **"📤 推送到设备"**
5. 等待推送完成
6. 验证：
   - ✅ 弹出成功对话框，提示 multi_media 进程已重启
   - ✅ 新配置已生效

### 4. 测试RTSP流配置（无IP输入）

1. 切换到 **"视频源管理"** 标签页
2. 在"RTSP流配置"区域，验证：
   - ✅ "设备IP"字段显示为只读的绿色IP地址（来自一键配置）
   - ✅ 没有IP输入框
3. 勾选需要的通道（通道0和/或通道1）
4. 点击 **"连接RTSP流"**
5. 验证连接成功

### 5. 验证WiFi配置已移除

1. 检查 **"算法控制"** 标签页
   - ✅ 没有WiFi配置区域
2. 检查菜单栏 **"工具"**
   - ✅ 没有"WiFi配置"选项
3. 如需配置WiFi，使用工具栏的 **"🚀 一键配置设备"** 按钮

### 6. 验证进程自动重启

#### 测试模型推送后重启
1. 推送一个新模型
2. 观察成功对话框
3. 验证包含：**"✅ multi_media进程已自动重启"**

#### 测试配置推送后重启
1. 推送一个配置文件
2. 观察成功对话框
3. 验证包含：**"✅ multi_media进程已自动重启，新配置已生效"**

## 常见问题排查

### Q1: 模型列表不显示删除按钮？
**解决**：确保已通过"一键配置设备"连接设备，然后点击"刷新模型列表"

### Q2: RTSP页面显示"未连接设备"？
**解决**：先通过"一键配置设备"完成设备连接

### Q3: 找不到WiFi配置入口？
**解决**：WiFi配置已整合到"一键配置设备"功能中，请使用工具栏的"🚀 一键配置设备"按钮

### Q4: 删除模型失败？
**解决**：
- 检查设备是否已连接
- 检查SSH连接是否正常
- 查看日志输出了解具体错误信息

### Q5: 推送后multi_media重启失败？
**解决**：
- 手动点击"算法控制"标签页中的"重启multi_media进程"按钮
- 检查设备网络连接
- 查看日志了解重启失败的具体原因

## 日志查看

所有操作都会记录到日志文件中：
- 日志位置：项目根目录下的 `logs` 文件夹
- 查看方式：菜单栏 **"工具" → "查看运行日志"**
- 日志内容：包含所有操作的详细记录和错误信息

## 性能监控

推送模型或配置后，可以切换到 **"性能监控"** 标签页：
1. 设置采样间隔（默认2秒）
2. 设置DDR频率（默认1848 MHz）
3. 点击 **"开始监控"**
4. 观察NPU、CPU、内存占用情况
5. 验证 multi_media 进程正常运行

## 成功标志

✅ 所有功能正常工作时应满足：
- 设备IP正确显示在各个相关页面
- 模型可以正常推送、刷新、删除
- 配置可以正常加载、编辑、推送
- RTSP流可以正常连接
- 每次推送后 multi_media 进程自动重启
- 没有WiFi配置的独立入口（已整合到一键配置）

## 反馈问题

如果遇到问题，请：
1. 查看日志文件了解详细错误信息
2. 检查设备网络连接
3. 确认设备SSH服务正常运行
4. 记录操作步骤和错误信息以便排查
