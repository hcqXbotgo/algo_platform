# RTMP直播功能使用指南

## 🎥 功能概述

新增的RTMP直播功能可以让您通过设备实时推流到RTMP服务器，支持720P和1080P两种画质。

---

## 🚀 快速开始

### 步骤1：连接设备

点击工具栏的 **"🚀 一键配置设备"** 按钮，完成设备配置。

### 步骤2：切换到"RTMP直播"标签页

在主界面顶部，点击 **"RTMP直播"** 标签。

### 步骤3：配置RTMP服务器

在"RTMP服务器配置"区域：
- **服务器地址**: 输入RTMP服务器URL（例如：`rtmp://192.168.17.108/live/test`）
- **画质选择**: 选择推流画质
  - `720P (流畅)` - 适合网络带宽有限的场景
  - `1080P (高清)` - 提供高清画质

### 步骤4：开始直播

点击 **"▶️ 开始直播"** 按钮

系统会自动：
1. 生成推流配置文件
2. 发送MQTT命令启动推流
3. 更新状态显示

### 步骤5：停止直播

点击 **"⏹️ 停止直播"** 按钮即可停止推流。

## 🔧 技术实现

### 开始推流命令

系统会依次执行以下命令：

```bash
# 1. 生成live_start.bin文件（填充RTMP URL）
printf '%-512s' 'rtmp://192.168.17.108/live/test' | tr ' ' '\0' > live_start.bin

# 2. 添加通道标识
printf '\x01' >> live_start.bin  # 1080P用\x01，720P用\x00

# 3. 发送MQTT命令启动推流
mosquitto_pub -t "IB0R" -f live_start.bin  # 720P
# 或
mosquitto_pub -t "IB1R" -f live_start.bin  # 1080P
```

### 停止推流命令

```bash
mosquitto_pub -t "IBKR" -f /oem/usr/bin/1.bin
```

## 💡 使用场景

### 720P画质
- ✅ 网络带宽有限
- ✅ 移动设备观看
- ✅ 长时间监控
- ✅ 多路同时推流

### 1080P画质
- ✅ 高清视频需求
- ✅ 专业直播场景
- ✅ 需要后期剪辑
- ✅ 网络条件良好

---

## ⚠️ 注意事项

1. **网络连接**
   - 确保设备已连接到网络
   - 确保可以访问RTMP服务器
   - 检查防火墙设置

2. **带宽要求**
   - 720P: 约需2-4 Mbps上行带宽
   - 1080P: 约需5-8 Mbps上行带宽

3. **设备性能**
   - 推流会占用一定的CPU和网络资源
   - 建议不要同时开启多路1080P推流

4. **RTMP服务器**
   - 确保RTMP服务器正在运行
   - 确认推流密钥/路径正确
   - 测试服务器可访问性

## 🔍 故障排查

### Q1: 推流启动失败

**可能原因：**
- 设备未连接
- RTMP服务器地址错误
- 网络不通

**解决方法：**
```bash
# 测试RTMP服务器连通性
ping 192.168.17.108

# 检查设备网络
adb shell ping 192.168.17.108
```

### Q2: 推流卡顿

**可能原因：**
- 网络带宽不足
- 画质设置过高

**解决方法：**
- 切换到720P画质
- 检查网络使用情况
- 降低其他网络活动

### Q3: 无法停止推流

**解决方法：**
```bash
# 手动发送停止命令
adb shell mosquitto_pub -t "IBKR" -f /oem/usr/bin/1.bin
```

## 🎯 最佳实践

### 推流前检查清单
- [ ] 设备已通过"一键配置"连接
- [ ] RTMP服务器地址正确
- [ ] 网络连接正常
- [ ] 选择合适的画质

### 优化建议
1. **首次使用** - 先用720P测试，确认稳定后再尝试1080P
2. **多路推流** - 建议使用不同端口或路径
3. **长时间推流** - 定期检查设备温度和性能
4. **重要直播** - 提前测试并准备备用方案
