# 三级自动连接功能说明

## 🎯 功能概述

现在算法验证上位机支持**真正的零感知自动连接**，启动程序后无需任何手动操作即可连接设备。

## ✨ 三级自动连接流程

### 第一级：SSH直连（最快速 ⚡）
```
启动程序
  ↓
读取 device_config.json
  ↓
尝试SSH连接保存的设备IP
  ↓
连接成功？
  ├─ 是 → 更新UI，自动连接MQTT，显示成功提示 ✅
  └─ 否 → 进入第二级
```

**耗时**: 约1秒  
**适用场景**: 设备在线，电脑和设备在同一网络

---

### 第二级：WiFi自动重连（智能备用 🔄）
```
检查是否保存了WiFi信息
  ↓
有WiFi信息？
  ├─ 是 → 通过ADB+WiFi自动重新配置设备网络
  │        ↓
  │       配置成功？
  │         ├─ 是 → 更新UI，自动连接MQTT，显示成功提示 ✅
  │         └─ 否 → 进入第三级
  └─ 否 → 进入第三级
```

**耗时**: 约30-60秒  
**适用场景**: 设备重启后WiFi断开，需要重新连接

---

### 第三级：提示用户一键配置（最终方案 ）
```
弹出对话框
  ↓
显示错误信息 + 提供两个选项：
  ├─ "是" → 清除旧配置，自动打开一键配置向导
  ─  "否" → 保留配置，用户可以稍后手动重试
```

**适用场景**: 设备IP变更、网络环境变化、首次使用

---

## 📁 配置文件结构

`device_config.json`（项目根目录）：

```json
{
  "device_ip": "192.168.17.181",
  "wifi_ssid": "XBOTGO-5G",
  "wifi_password": "your_password",
  "rtsp_0": "rtsp://192.168.17.181/live/0",
  "rtsp_1": "rtsp://192.168.17.181/live/1",
  "last_connected": "2026-05-18 18:30:45"
}
```

### 字段说明
| 字段 | 说明 | 是否必须 |
|------|------|---------|
| `device_ip` | 设备IP地址 | ✅ 必须 |
| `wifi_ssid` | WiFi名称 | ⚠️ 可选（用于第二级重连） |
| `wifi_password` | WiFi密码 | ️ 可选（用于第二级重连） |
| `rtsp_0` | RTSP通道0地址 | ✅ 必须 |
| `rtsp_1` | RTSP通道1地址 | ✅ 必须 |
| `last_connected` | 最后连接时间 | ✅ 自动记录 |

## 🚀 使用指南

### 首次配置
1. **确保设备已通过USB连接**
2. 点击工具栏 **"🚀 一键配置设备"**
3. 输入WiFi信息（SSID和密码）
4. 等待配置完成（约1-2分钟）
5. 配置成功后会显示：
   - ✅ 配置已保存，下次启动时将自动连接
   - 设备IP、RTSP地址、MQTT状态

### 后续启动（零感知体验）
1. **直接启动程序**
2. 观察状态栏变化：
   - `正在自动连接设备: 192.168.17.181...`
   - 约1秒后显示 `已自动连接到设备: 192.168.17.181`
3. 弹出"自动连接成功"对话框
4. **现在可以直接使用所有功能了！**

### 切换设备
1. 点击菜单栏：**文件 → 清除设备配置**
2. 确认对话框
3. 点击 **"🚀 一键配置设备"** 配置新设备

### 网络环境变化
如果设备IP变更或网络环境变化：
1. 启动程序时会尝试自动连接
2. 如果失败，会弹出提示对话框
3. 选择"是"清除旧配置
4. 自动打开一键配置向导
5. 重新输入WiFi信息并配置

## 🔧 技术实现

### 核心方法

#### 1. `auto_connect_device()`
- **功能**: 三级自动连接主入口
- **调用时机**: 程序启动后500ms
- **流程**: SSH直连 → WiFi重连 → 提示用户

#### 2. `_wifi_reconnect_attempt()`
- **功能**: WiFi自动重连（后台线程）
- **线程安全**: 使用独立线程，避免阻塞UI
- **回调机制**: 使用 `QTimer.singleShot(0, partial(...))` 确保UI更新在主线程

#### 3. `_on_auto_connect_success()`
- **功能**: 连接成功后的UI更新
- **操作**: 
  - 更新状态栏显示
  - 更新RTSP地址
  - 自动连接MQTT
  - 显示成功对话框

#### 4. `save_device_config()`
- **功能**: 保存设备配置（包括WiFi信息）
- **调用时机**: 一键配置成功后
- **存储格式**: JSON

#### 5. `clear_device_config()`
- **功能**: 清除保存的设备配置
- **调用时机**: 用户通过菜单选择
- **副作用**: 重置所有连接状态UI

### 关键技术点

1. **延迟启动**: 使用 `QTimer.singleShot(500, ...)` 确保UI初始化完成
2. **后台线程**: WiFi重连在独立线程执行，符合长耗时操作规范
3. **主线程安全**: 所有UI更新通过 `QTimer.singleShot(0, ...)` 调度到主线程
4. **参数绑定**: 使用 `functools.partial` 避免lambda闭包问题
5. **详细日志**: 所有操作都记录日志，便于问题排查
